MODULE 7 BLUEPRINT - OBSERVABILITY & AUDIT
===========================================
Phase 4 scope (with hardening). Know what CORE is doing without SSH.
Three independent strands: metrics, audit, structured logs. Nothing here
sits on a request hot path except one-line metric increments.

----------------------------------------------------------------------
A. UNITS - package dev.shayveri.core.observability
----------------------------------------------------------------------

V1. Build additions (consumed dependencies, build.gradle.kts):
    runtimeOnly("io.micrometer:micrometer-registry-prometheus")
        - turns /actuator/prometheus on (endpoint already exposed in
          application.yml since Phase 0; without this dep it 404s).
    implementation("net.logstash.logback:logstash-logback-encoder:7.4")
        - JSON log encoding for V6.

V2. ShayveriMetrics                 (the one metrics entry point)
    Thin named wrappers so modules never sprinkle raw registry calls:
      telemetryAccepted()        counter shayveri_telemetry_ingest_total
      openCloudOutcome(ok)       counter shayveri_opencloud_calls_total{outcome}
      jobTransition(from, to)    counter shayveri_job_transitions_total{...}
    Gauges registered at construction:
      shayveri_queue_depth{type} - Supplier pulling from QueueStore
        (Micrometer polls the lambda on scrape - the gauge costs nothing
        between scrapes)
      shayveri_ws_sessions - Supplier from SimpUserRegistry.getUserCount()
    Call sites: one line each in TelemetryService (A7), EgressService
    (E4), JobService (J8) - added when this module lands, through THIS
    unit only (same discipline as RealtimePublisher: modules call ours,
    never the framework's).
    Consumes: MeterRegistry - counter(name, tags...).increment();
    Gauge.builder(name, supplier).register(registry). Micrometer is
    already on the classpath via Actuator since Phase 0.

V3. AuditRecord                     (Mongo document)
    Collection: audit. Fields: @Id; Instant at; String who (api key
    LABEL - the ApiKeyResolver principal, "dash" today, "averi" when
    keys go per-person); String action ("config.activate",
    "job.create"...); String target; Map before; Map after. No TTL -
    "who changed the spawn rate" must have an answer in March.
    Consumes: @Document, @Id, @Indexed(at).

V4. AuditStore (interface) + V5. MongoAuditStore (+ repository)
    record(AuditRecord), query(range, action) - the established rule-5
    pattern, last time.

V6. AuditService
    audit(who, action, target, before, after) - fire-and-forget on the
    virtual executor (an audit write must never slow or fail a
    mutation; failure logs loudly instead).
    Call sites (the plan's law: EVERY DASH mutation): ConfigService
    save+activate (O8), JobService create (J8), future key management.
    Consumes: @Service, Executor.

V7. logback-spring.xml              (src/main/resources)
    Structured JSON to stdout via LogstashEncoder; plain pattern kept
    on a "dev" Spring profile so bootRun stays human-readable.
    Consumes: logback config convention (Spring Boot picks the file up
    by name), LogstashEncoder class from V1's dependency.

----------------------------------------------------------------------
B. TEST PLAN
----------------------------------------------------------------------
T1. Acceptance: GET /actuator/prometheus (authenticated) contains
    shayveri_telemetry_ingest_total after one telemetry POST, and
    shayveri_queue_depth after one job enqueue.
T2. Acceptance: activate a config version -> exactly one audit document
    (action=config.activate, before=old version, after=new, who=label).
T3. Audit resilience: AuditStore that throws -> the mutation still
    succeeds (200), error logged.
T4. Log shape (cheap smoke): with prod profile, one log line parses as
    JSON with level+logger+message keys.
